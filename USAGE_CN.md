# OctoFinance V2 使用指南

## 界面布局

打开 http://localhost:5173 后，界面分为三个区域：

```
┌─────────────────────────────────────────────────────────┐
│  StatusBar: 用户名 · 组织数 · AI状态 · [Sync Data]按钮  │
├──────────────┬──────────────────────────────────────────┤
│              │                                          │
│   左侧边栏    │            主聊天区域                     │
│              │                                          │
│  Overview    │   欢迎页 / AI 对话                        │
│  (概览数据)   │                                          │
│              │   [Quick Prompt 快捷按钮]                  │
│  Orgs        │                                          │
│  (组织列表)   │   用户消息 ↔ AI回复                       │
│              │                                          │
│  Pending     │                                          │
│  Actions     │                                          │
│  (待审批操作)  │   ┌─────────────────────────────┐       │
│              │   │  输入框  [Clear] [Send/Stop] │       │
│              │   └─────────────────────────────┘       │
└──────────────┴──────────────────────────────────────────┘
```

## 第一步：检查状态

看顶部 **StatusBar**：

- 绿点 + 用户名 + org 数量 → 后端已连接，数据已同步
- 绿点 + "AI Ready" → Copilot AI 引擎就绪，可以开始对话
- 黄点 + "AI Starting..." → AI 引擎还在启动，等几秒

如果显示异常，点右侧 **Sync Data** 按钮手动触发数据同步。

## 第二步：查看左侧概览

左侧边栏自动显示：

- **Overview**: 总席位数、活跃数、不活跃数、利用率、月度成本、月度浪费
- **Organizations**: 所有自动发现的组织，每个显示计划类型和席位情况
- **Pending Actions**: AI 生成的待审批操作建议（初始为空）

## 第三步：和 AI 对话

这是核心功能。在主区域的输入框里用自然语言提问，中英文都行。

### 快捷按钮（首次进入可见）

| 按钮 | 作用 |
|------|------|
| **Overview** | 全局 Copilot 使用和成本概览 |
| **Inactive Users** | 查找 30 天以上未使用 Copilot 的用户 |
| **Cost Optimization** | 分析浪费并给出优化建议 |
| **ROI Analysis** | 计算 Copilot 投资回报率 |

点任意一个即可开始。

### 常用对话示例

**查看概览：**
```
帮我看看当前所有组织的 Copilot 使用情况和成本
```

**找不活跃用户：**
```
哪些用户超过30天没用过 Copilot？浪费了多少钱？
```

**深入某个组织：**
```
nekoaru 组织的 Copilot 利用率怎么样？哪些人用得多哪些人不用？
```

**成本优化：**
```
帮我分析一下哪里可以省钱，给出具体建议
```

**ROI 分析：**
```
我们的 Copilot 投资值不值？每个活跃用户的成本是多少？
```

**创建操作建议：**
```
帮我把不活跃的用户都列出来，创建移除席位的建议
```

**跨组织对比：**
```
对比一下各个组织的 Copilot 利用率，哪个组织浪费最严重？
```

**多轮对话（AI 会记住上下文）：**
```
你: 帮我看看 nekoaru 的不活跃用户
AI: 找到3个不活跃用户...需要我创建移除建议吗？
你: 好的，帮我创建
AI: 已创建建议，预计节省$117/月。需要执行吗？
你: 先不执行，我再想想
```

## 第四步：处理 AI 建议

当 AI 创建了操作建议后，左侧 **Pending Actions** 区域会出现卡片，显示：

- 操作类型（如 "remove seats"）
- 涉及的组织和用户
- 预估节省金额
- **Approve & Execute** 按钮 → 确认执行（会真的调 GitHub API 移除席位）
- **Reject** 按钮 → 拒绝此建议

> ⚠️ **Approve & Execute 会真正执行操作**（如移除用户的 Copilot 席位），请确认后再点。

## 对话过程中的 Tool 标识

AI 回复时，你可能看到消息上方出现小标签：

- ⏳ `get_all_seats` → AI 正在读取席位数据
- ⏳ `find_inactive_users` → AI 正在分析不活跃用户
- ⏳ `get_cost_overview` → AI 正在计算成本
- ⏳ `record_recommendation` → AI 正在创建操作建议
- ✅ 标签变绿 → 该 tool 执行完成

这些标签让你知道 AI 在用哪些工具分析数据，不需要你操作。

## 其他操作

| 操作 | 方法 |
|------|------|
| 手动同步数据 | 点顶部 **Sync Data** 按钮 |
| 清空对话 | 点输入框左侧 **Clear** 按钮 |
| 中断 AI 回复 | AI 回复中点 **Stop** 按钮 |
| 切换语言 | 直接用中文或英文提问，AI 会自动匹配 |
